# RepoLens

Open-source projects rarely lack possible work; they lack a trustworthy way to
identify which work matters and where to begin. RepoLens turns a public GitHub
repository into an evidence-backed readiness audit and contribution queue.

It checks community health, setup, tests, CI, maintainability, selected
frontend quality signals, and static source relationships. Findings cite exact
repository evidence, disclose uncertainty, and become copyable contribution
tasks. RepoLens also names files with zero detected inbound references while
warning users to verify runtime and framework loading before removal.

The target repository is never executed. Static interface reconstruction and
the architecture explorer remain available as supporting evidence, while the
main experience is designed around decisions and next actions.
