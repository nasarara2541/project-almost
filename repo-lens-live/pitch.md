# RepoLens — Pitch and Business Case

## One-line pitch

RepoLens lets developers see a web application running and understand exactly which code produces what they are looking at.

## Problem

When developers inherit an unfamiliar frontend, they face two separate tasks: figuring out what the application does and figuring out where that behavior lives in the code. Existing code browsers show files, while live demos show screens; neither connects the two.

## Solution

RepoLens combines a temporary live preview with architecture analysis and AI-powered code tracing. A developer can move from a visible screen to the responsible route, component, API, and source files.

## Demo hook

Open a real repository, click a visible feature, and watch RepoLens explain the path from the screen to the code.

## Users

- Engineering teams onboarding developers
- Open-source contributors
- Technical leads reviewing inherited applications
- Educators teaching software architecture

## Business model

Freemium: public small repositories are free; private repositories, larger limits, persistent workspaces, team collaboration, and private deployment are paid features.

## Why it is different

It connects three layers in one experience:

```text
Visible product → Architecture → Source code
```
